# Neural-networks-from-scratch

#### Курсовая работа "Нейронные сети с нуля"

# Структура проекта

- `/pkg` - внешние библиотеки
    - `/pkg/AnyObject` - реализация стирающего типа (интерфейса).
    - `/pkg/Catch2` - библиотека для тестирования. Подключена как github submodule.
    - `/pkg/Eigen` - библиотека для линейной алгебры. Подключена как github submodule.

- `/src` - исходный код библиотеки

- `/tests` - тесты (подробнее в соответствующем разделе)

Также есть стайлгайд в виде файла `clang-format` и скрипт `format.sh` применяющий его к проекту.

# Сборка

Проект собирается при помощи `CMake`

```bash
mkdir build/ && cd build/
cmake ..
cmake --build .
```


# Документация

## Общие сведения

Весь код лежит внутри `namespace NeuralNetwork`

## Основные класcы/сущности

Все примеры использования (в том числе задания пользовательских функций/классов) можно найти в unit-тестах.

### Класс LossFunction

Класс `LossFunction` предназначен для вычисления функций потерь и их градиентов. Поддерживает стандартные функции потерь :

- `Mse` - Среднеквадратическая ошибка(Mean Squared Error)
- `CrossEnthropy` - Перекрестная энтропия


Класс поддерживает пользовательские функции. Чтобы использовать функции `user_loss` и `user_loss_gradient` в качестве функции потерь и градиента функции потерь они должны иметь сигнатуры:

```cpp
double user_loss(const Vector&, const Vector&);
RowVector user_loss_gradient(const Vector&, const Vector&);
```

Чтобы создать экземпляр класса с такими функциями достаточно вызвать конструктор и передать их в него.

### Класс Layer

Отвечает за слой нейронной сети. Хранит в себе один из классов:
- `LinearLayer`
- `NonLinearLayer`

Имеет 2 метода:

- `Forward(x)` - вычисляет значение функции активации для вектора $x$

- `Backward(x, u)` - вычисляет градиент функции активации для входного вектора $x$, учитывая градиент предыдущего слоя $u$.

##### LinearLayer

Линейный слой нейронной сети, для входящего вектора $x$ применяет функцию $A x + b$, где $A$ - матрица, $b$ - вектор.


##### NonLinearLayer

Нелинейный слой нейронной сети (хранит функцию активации). Поддерживает стандартные функции активаций:

- `Sigmoid`
- `ReLU`
- `LeakyReLU`
- `Softmax`

Класс поддерживает пользовательские функции. Чтобы использовать функции `user_forward` и `user_backward` в качестве функции активации и её градиента они должны иметь сигнатуры:

```cpp
Vector user_forward(Vector const&);
RowVector user_backward(Vector const&, RowVector const&);
```

Чтобы создать экземпляр класса с такими функциями нужно вызвать конструктор и передать их в него.

### Класс Optimizer

##### Вспомогательный класс `Optimizers::Cache`

Используется для хранения кэшей. Хранит в себе один из классов:

- `Optimizers::Caches::Empty`
- `Optimizers::Caches::SimpleState`
- `Optimizers::Caches::Adam`

Каждый из вариантов класса имеет конструктор принимающий размеры вектора $x$ и вектора градиента $u$, которые соотвествуют размерам для которых работает `Optimizer`

##### Сам класс `Optimizer`

Хранит в себе класс оптимизирующий нахождение градиента функции. Не хранит в себе необходимые для работы оптимизируещего классы данные, делигирует их хранение классу `Optimizers::Cache`.

Имеет 2 метода:

- `Optimize(cache, x, u, learning_rate)` - принимает кэш, вектор $x$, градиент предыдущего слоя $u$ и гиперпараметр $learning\_rate$ (скорость обучения). Возвращает вычисленный градиент.

- `ConstructRequiredCache(x_size, u_size)` - принимает размеры вектора $x$ и вектора градиента предудщего слоя $u$. Создает чистый кэш соотвествующий классу лежащему внутри.

Поддерживает хранение стандартных классов:
- `Optimizers::Classic`
- `Optimizers::Momentum`
- `Optimizers::AdaGrad` 
- `Optimizers::RMSProp` 
- `Optimizers::Adam`

Все они вычисляют градиент (алгоритм вычисления соотвествует названию класса).

Поддерживает пользовательские классы. Чтобы положить пользовательский класс `user_optimizer` внутрь этого класса, `user_optimizer` должен реализовывать метод `Optimize` со следующей сигнатурой:

```cpp
Gradients Optimize(Optimizers::Cache& cache, const Vector& x, const RowVector& u, double learning_rate) const;
```

И определять внутри себя через `using RequiredCacheType` какой кэш он требует, например:

```cpp
class user_optimizer {
    ...
    using RequiredCacheType = Optimizers::Caches::SimpleState;
    ...
}
```


### Класс DataLoader

Отвечает за загрузку датасетов. В конструкторе принимает пользовательскую функцию с сигнатурой `DataLoader::LoadFunctionType` и 2 пути до файлов с парметрами(lables) и значениями(targets) датасета. Имеет 1 метод:

- Load() - применяет пользовательскую функцию для загрузки датасета

### Класс Network

Основной класс, отвечает за нейронную сеть. Хранит функцию потерь, слои, оптимайзер, кэши оптимайзера и гиперпареметр $learning\_rate$. Имеет основные методы:

- `TrainSingle(data_sample)` - тренирует модель на одном примере данных

- `TrainSingle(data_batch)` - тренирует модель на массиве данных

- `Predict(x)` - предсказывает значение на векторе $x$

- `GetWeights()` - возвращает все веса (параметры) модели

- `CheckAccuracy(data, accuracy_funcion)` - вычисляет Accuracy (долю верных предсказаний) на датасете `data` пользуясь предикатом `accuracy_funcion` (выдаёт 1 если предсказание верное, 0 - иначе)

# Тестирование

### Unit-тесты

Unit тесты находятся в директории `/tests/unit`. Для каждого основного класса собирается бинарник с unit-тестами. Также есть `test_all` - для тестирования сразу всех. Полный список:
- `test_all`
- `test_data_loader`
- `test_layer`
- `test_loss`
- `test_network`
- `test_optimizer`

Тестирование определенной цели:
```bash
./tests/unit/test_layer  # in /build
```

Или же через ctest сразу все:

```bash
ctest  # in /build
```

### Датасет MNIST

Есть возможность тестировать на датасете MNIST:

```bash
./tests/MNIST/test_mnist  # in /build
```

Обратите внимание что сам датасет для этого нужно скачать и указать пути к данным в файле `/tests/MNIST/test_mnist.cpp`
